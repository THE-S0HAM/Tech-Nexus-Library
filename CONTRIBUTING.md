# Contributing to Tech-Nexus

Thanks for your interest in contributing to Tech-Nexus. We welcome clear, well-sourced contributions that follow the repository's structure.

How to contribute
- Fork the repository and create a branch: `git checkout -b topic/your-article-name`
- Add your article as a Markdown file under the appropriate folder (for example, `01-DevOps-Engineering/Kubernetes/your-article.md`).
- Use the article template in `TEMPLATES/article_template.md` for frontmatter and structure.
- Commit with a clear message and open a pull request.

Article guidelines
- Include frontmatter at the top of each article:

```
---
title: "Your article title"
category: "DevOps"
tags: [k8s, containers]
last_updated: 2025-11-29
---
```
- Keep articles factual, vendor-neutral when possible, and cite sources.
- Use headings, code blocks, and diagrams where helpful.

Style and formatting
- Use Markdown and follow consistent heading levels.
- Prefer clear, actionable steps and examples.

License & attribution
- By contributing you agree to license your contribution under this repository's MIT license.
